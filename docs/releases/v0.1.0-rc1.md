# AgentOS v0.1 RC1 — Harness + single-project orchestration hardening

This release-candidate checkpoint preserves the first known-good AgentOS v0.1
state after the staged hardening PR stack and RC dogfood fixes landed.

## Included

- Stage 0 characterization tests and fixtures.
- Full-by-default `npm run agent-check` with structure-only opt-out.
- Strict workflow validation through `agent-os workflow validate --strict`.
- Trust-mode defaults, Codex App Server event policy, and strict
  `codex-doctor`.
- Redacted logs, prompts, events, handoffs, and run artifacts.
- Schema-versioned issue state, run summaries, review artifacts, and validation
  evidence.
- Durable per-run artifact directories with artifact hashes and
  `runs inspect` hash warnings.
- Multiple PR metadata support in issue state, including lazy legacy `prUrl`
  migration.
- Atomic workspace locks.
- Upsertable Linear lifecycle comments.
- Local-only simulation and replay support that does not call Linear, GitHub, or
  Codex.
- Hardened single-project Linear orchestration for one configured repository.

RC1 covers hardened single-project orchestration. It does not cover true
multi-project daemon scheduling.

## Non-Goals

RC1 intentionally does not include:

- Stage 9A durable retry/startup reconstruction implementation.
- True multi-project daemon scheduling.
- GitHub merge queue or protected-branch automation.
- npm package publication.
- Broad architecture rewrites.
