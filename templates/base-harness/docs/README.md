# Project Docs

This directory is the source of truth for agent-assisted development.

- `product/` explains what the project is and what users need.
- `architecture/` explains boundaries, layers, and integration points.
- `decisions/` records durable design decisions.
- `quality/` defines validation expectations.
- `security/` documents security-sensitive areas.
- `runbooks/` holds operational steps agents and humans can follow.
- `generated/` is reserved for generated reports and artifacts.
